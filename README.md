# paper-factor

Extracts and generates quantitative trading factors from academic papers using LLMs.

## Quick Start

```bash
git clone https://github.com/whzzzz2004-netizen/paper-factor.git
cd paper-factor
bash scripts/setup.sh
```

Setup handles everything: dependency install, config, Docker image build, data download.

Then edit `.env` with your API keys, drop a paper in `papers/inbox/`, and run:

```bash
start --report-file papers/inbox/paper.pdf
```

Outputs go to `git_ignore_folder/factor_outputs/`.

## Requirements

- Python 3.10+
- Docker

## CLI Commands

| Command     | Description |
|-------------|-------------|
| `start`     | Full pipeline — extract + generate factors (Docker) |
| `run`       | Same pipeline, no forced backend |
| `sync`      | Download arXiv papers by search query |
| `init`      | Initialize workspace and download market data |
| `dashboard` | Launch factor results dashboard |

## Data Structure

Market data is stored as per-stock parquet files:

```
git_ignore_folder/factor_implementation_source_data/stock_data/
├── daily/          # 500 stocks × 1005 days
│   ├── SH600000.parquet   # DatetimeIndex, columns: open/close/high/low/volume/factor/market_cap/industry_sw/turnover
│   ├── stock_list.json
│   └── trade_dates.json
└── minute/         # 500 stocks × 1005 days × 240 bars/day
    ├── SH600000.parquet   # DatetimeIndex, columns: open/high/low/close/volume/vwap
    ├── stock_list.json
    └── trade_dates.json
```

Generated by `scripts/generate_stock_data.py`.

## Factor Templates

The pipeline supports 4 factor templates, automatically selected based on factor type:

| Template | Function | Use Case |
|----------|----------|----------|
| A: Single Stock | `calc_factor_single_stock(df, trade_date)` | Momentum, reversal, MA, etc. |
| B: Cross Section | `calc_factor_cross_section(all_data, trade_date)` | Industry ranking, sector heat, etc. |
| C: Deep Learning | `train_model(all_data, date)` + `predict(model, df, date)` | GRU, TCN, LSTM, etc. |
| D: Minute | `calc_factors_one_day(df)` | Intraday factors |

The AI only fills in the function body. Framework code (data loading, parallel execution, wide-table output) is automatically injected.

## Factor Type Classification

When extracting factors from papers, the AI classifies each factor as:
- `single_stock`: Only needs one stock's data
- `cross_section`: Needs multiple stocks' data
- `deep_learning`: Needs neural network training

Classification happens via a separate LLM call after factor extraction.

## Output Format

Factors are exported as wide-table parquet files:

```
Code        SH600000  SH600001  SH600002  ...
Date
2021-05-24  0.05      0.03      -0.02     ...
2021-05-25  0.02      -0.01     0.04      ...
```

- Index: `Date` (DatetimeIndex)
- Columns: `Code` (stock codes)
- Missing values: NaN
